- Phase 2 shopify-app gotchas:
  - This template uses **Polaris web components** (`s-page`, `s-section`, `s-button`, `s-box`, `s-badge`, `s-text-field`, ...), NOT React `@shopify/polaris`. There is no `@shopify/polaris` dependency. Build admin UI with `s-*` elements + App Bridge (`useAppBridge`, `shopify.toast.show`). Nav is `<s-app-nav><s-link>` in `app/routes/app.tsx`.
  - Embedded admin routes are session-gated: a raw `GET /app/*` returns HTTP 410 with an App Bridge bounce script (not the page). They only render inside admin.shopify.com for a logged-in merchant. Can't be curl'd or headlessly driven without a Shopify login. Verify admin business logic by calling the `app/services/*` functions directly against the DB (as done in Phase 2), not by hitting the routes.
  - `app/services/db.server.ts` MUST stay a re-export shim of `../db.server` (the app's PrismaClient shared with PrismaSessionStorage). Do NOT `new PrismaClient()` there — two clients = two pools on Neon.
  - Prisma: `package.json#prisma` seed hook works but warns it's deprecated (Prisma 7 wants `prisma.config.ts`). Migrations live in `prisma/migrations/` on Postgres now; `npm run setup` (`prisma migrate deploy`) is the prod path.

  - `shopify app dev` runs pre-dev `npx prisma generate` and reads `DATABASE_URL` from `shopify-app/.env`. App server (vite) prints its `Local:` port in the CLI output; the embedded app iframe is served from the current `*.trycloudflare.com` tunnel.
